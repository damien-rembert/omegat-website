Versiones heredadas

Aún se pueden encontrar versiones más antiguas que se han sustituido de OmegaT.

Las versiones heredadas son versiones más antiguas que se han sustituido. Para la mayoría de usuarios, carecen de interés. Sin embargo, algunos usuarios pueden encontrar restricciones (impuestas por sus sistemas operativos o versión de Java) que pueden hacer que prefieran una versión más antigua.

Con/sin JRE

Es necesario que tenga Java Runtime Environment (JRE) instalado en su equipo para que OmegaT funcione.

Si está seguro de que ya tiene una versión adecuada de JRE instalada en su equipo, puede descargar una versión de OmegaT sin JRE. La descarga será más rápida y la instalación necesitará menos espacio en su disco duro. En ese caso, no seleccione la versión predeterminada; seleccione una versión diferente (p. ej., la estándar o la última) y seleccione después una versión que no incluya Java.

Si instala la versión de OmegaT con JRE aunque ya tenga JRE instalado en su equipo, el JRE de OmegaT se instalará por separado y no entrará en conflicto. Por lo tanto, siempre es seguro instalar la versión de OmegaT con JRE. Si no está seguro de si tiene o no una versión adecuada de JRE, seleccione una versión con JRE (o la versión predeterminada).

Usuarios de Linux: la mayoría de distribuciones de Linux cuentan con Java Runtime Environment, pero algunas implementaciones de código abierto no funcionan bien con OmegaT (aunque no lo parezca en un primer momento). Si es relativamente nuevo en Linux, le recomendamos que seleccione una versión de OmegaT con JRE incluido. Los usuarios de Linux más experimentados pueden preferir instalar JRE de Oracle (antes Sun) a nivel de sistema.

Firmado/sin firmar

macOS tiene una función llamada Gatekeeper que incrementa la seguridad dificultando la ejecución de programas sin firmar. OmegaT para macOS está disponible en versión firmada y sin firmar; cuentan con las mismas funciones, pero es preferible usar la versión firmada porque el sistema reconoce que la ha publicado un desarrollador conocido.

Es posible que se tarde un tiempo en publicar las versiones firmadas. Los usuarios que tengan prisa pueden usar en su lugar la versión sin firmar; si tiene dificultades para ejecutar la aplicación, consulte cómo Abrir una app de un desarrollador no identificado.

La versión "master" (alfa) refleja el último estado de desarrollo. Se recomienda solo para desarrolladores y testers serios. También se puede conseguir usando el siguiente comando:

git clone https://git.code.sf.net/p/omegat/code omegat-code

Las compilaciones periódicas están disponibles en Sourceforge.

Los paquetes de código fuente no son necesarios para los usuarios y están pensados solo para desarrolladores. Se pueden encontrar en las carpetas de las versiones compiladas correspondientes.
